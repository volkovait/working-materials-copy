# Пример архитектуры Supervisor на LangGraph JS

Этот проект демонстрирует использование официального пакета `@langchain/langgraph-supervisor` для создания иерархической мультиагентной системы.

## Описание примера

В примере реализована классическая утилитарная связка для работы с локальным проектом:
- **Researcher (Исследователь)** — агент, имеющий доступ к чтению файловой системы (инструменты: `list_directory`, `read_file`). Он может просматривать директории и читать исходники проекта.
- **Writer (Писатель)** — агент, имеющий доступ к записи файлов (инструмент: `write_file`). Он генерирует код или документы и сохраняет их на диск.
- **Supervisor (Супервизор)** — главный маршрутизирующий агент, который принимает задачу от пользователя, решает, кому делегировать подзадачи, и контролирует процесс до финального результата.

**Пример задачи в коде:** проанализировать `package.json` текущего проекта и сгенерировать на его основе файл отчета `SUMMARY.md`.

## Как запустить

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Убедитесь, что в родительской директории `code/week-4/10-day/node` есть файл `.env` с указанными ключами (если его нет, скопируйте там `.env.example` в `.env`):
   ```bash
   OPENAI_API_KEY=sk-ваш-ключ
   ```

3. Запустите скрипт:
   ```bash
   npm start
   ```

В консоли будет выведен лог того, как супервизор передает управление между агентами. По итогу работы скрипта в папке появится сгенерированный файл `SUMMARY.md`.

## Преимущества библиотеки `@langchain/langgraph-supervisor`

- **Меньше шаблонного кода:** Библиотека берет на себя самую сложную часть маршрутизации и handoff-механизмов. Вам достаточно определить набор обычных React-агентов (через `createReactAgent`) и передать их в `createSupervisor()`. 
- **Гибкость:** Вы можете строить многоуровневые иерархии (супервизор управляет другими супервизорами).
- **Контроль истории:** Можно настраивать, будет ли в общую историю передаваться полная переписка агента (`outputMode: "full_history"`) или только его финальный ответ (`outputMode: "last_message"`).
