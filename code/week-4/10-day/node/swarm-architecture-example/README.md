# Архитектура Swarm (Децентрализованное сотрудничество)

В этом примере демонстрируется мультиагентный подход **"Swarm"** (Рой) с использованием библиотеки LangGraph на Node.js.

## Концепция архитектуры

В архитектуре Swarm агенты работают в едином контексте сообщений (Shared Scratchpad), где они "видят" все реплики друг друга. В отличие от архитектуры Supervisor, здесь **нет центрального менеджера**. Агенты передают управление друг другу напрямую (peer-to-peer), обмениваясь данными и результатами работы по мере необходимости.

Этот подход больше похож на мозговой штурм в реальной команде, где участники обсуждают идею, критикуют ее и дорабатывают до тех пор, пока не придут к консенсусу.

## Структура проекта

Основной код находится в файле `index.js`. Процесс организован в виде графа состояний (`StateGraph`):

### Состояние графа (`GraphState`)
В данном подходе состояние максимально простое:
- `messages`: единый массив сообщений, который пополняется каждым агентом. Эта общая "доска" позволяет каждому участнику видеть всю историю диалога.

### Узлы (Nodes)
1. **Brainstormer (Идейщик)**: Отвечает за генерацию первоначальных идей или их улучшение на основе полученной критики. Отвечает коротко и по делу.
2. **Critic (Критик)**: Выступает в роли конструктивного рецензента. Оценивает предложенную идею. Если идея требует доработки, он пишет замечания. Если идея отличная и готова к использованию, он добавляет в ответ ключевое слово `FINAL_ANSWER`.

### Граф выполнения и маршрутизация
- **Точка входа (`START`)** направляет первоначальный запрос пользователя к **Brainstormer**.
- Brainstormer генерирует идею и напрямую передает управление узлу **Critic** (`addEdge("Brainstormer", "Critic")`).
- **Условный переход (`checkCompletion`)**: После ответа Критика проверяется наличие кодового слова `FINAL_ANSWER` в последнем сообщении:
  - Если слово найдено, процесс завершается (`END`).
  - Если слова нет, управление возвращается обратно к **Brainstormer** для доработки идеи на основе замечаний.

## Зависимости
- `@langchain/langgraph` — для построения графа агентов.
- `@langchain/openai` — для вызова языковой модели.
- `@langchain/core` — для работы со структурой сообщений.
- `dotenv` — для работы с переменными окружения.

## Как запустить

1. Установите зависимости проекта:
   ```bash
   npm install
   ```
2. Убедитесь, что в родительской директории (или в текущей) настроен файл `.env` с ключом:
   ```env
   OPENAI_API_KEY=ваш_ключ
   OPENAI_BASE_URL=опциональный_базовый_url
   ```
3. Запустите выполнение графа:
   ```bash
   npm start
   ```

В консоли будет выведен итеративный диалог: Идейщик будет предлагать варианты, а Критик — оценивать их до тех пор, пока идея не будет доведена до совершенства.
