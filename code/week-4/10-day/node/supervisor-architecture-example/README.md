# Архитектура Supervisor (Иерархия и Контроль)

В этом примере демонстрируется мультиагентный подход **"Supervisor"** (Супервизор/Менеджер) с использованием библиотеки LangGraph на Node.js.

## Концепция архитектуры

В архитектуре Supervisor один центральный агент (`Supervisor`) управляет процессом выполнения задачи. Он получает первоначальный запрос пользователя и маршрутизирует его между специализированными агентами (в данном случае `Researcher` и `Writer`). 

Исполнители выполняют узконаправленные задачи и всегда возвращают результат обратно менеджеру. Менеджер, анализируя обновленное состояние (историю сообщений), принимает решение:
- Передать задачу исследователю для сбора фактов.
- Передать задачу писателю для формирования итогового текста.
- Завершить процесс, если задача полностью выполнена (статус `FINISH`).

Такой подход позволяет жестко контролировать ход выполнения задачи и гарантировать, что агенты-исполнители не будут отклоняться от своей основной функции.

## Структура проекта

Основной код находится в файле `index.js`. Процесс строится на базе `StateGraph`, который хранит состояние выполнения:

### Состояние графа (`GraphState`)
Состояние содержит:
- `messages`: массив всех сообщений (история переписки между агентами и пользователем).
- `next`: инструкция маршрутизации, указывающая, кто должен работать следующим.

### Узлы (Nodes)
1. **Supervisor (Менеджер)**: Не генерирует контент. Использует структурированный вывод (structured output через `zod`), чтобы выбрать следующий шаг: `"Researcher"`, `"Writer"` или `"FINISH"`.
2. **Researcher (Исследователь)**: Собирает факты и формирует краткие тезисы по запросу пользователя. Возвращает новые сообщения в общую историю.
3. **Writer (Писатель)**: Берет собранные факты и пишет на их основе связный и красивый текст, заданного стиля.

### Граф выполнения
- **Точка входа (`START`)** всегда направляет поток к Supervisor.
- Supervisor использует **условные переходы (`addConditionalEdges`)** для маршрутизации задачи к конкретному исполнителю или к завершению (`END`).
- Исполнители (Researcher и Writer) после завершения своей работы **всегда возвращают поток обратно** к Supervisor.

## Зависимости
- `@langchain/langgraph` — для построения графа состояний.
- `@langchain/openai` — для работы с языковой моделью.
- `@langchain/core` — для базовых абстракций (сообщения).
- `zod` — для строгого типизирования вывода Supervisor.

## Как запустить

1. Установите зависимости проекта:
   ```bash
   npm install
   ```
2. Убедитесь, что в родительской директории (или в текущей) есть файл `.env` с ключом API. Пример:
   ```env
   OPENAI_API_KEY=ваш_ключ
   OPENAI_BASE_URL=опциональный_базовый_url
   ```
3. Запустите скрипт:
   ```bash
   npm start
   ```

В консоли вы увидите пошаговый процесс маршрутизации и ответы каждого из агентов.
