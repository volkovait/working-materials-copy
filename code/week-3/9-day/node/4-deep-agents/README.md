# deep-agents — Deep Agents (планирование, инструменты, HITL)

Демонстрационный проект на базе **Deep Agents** (библиотека [deepagents](https://www.npmjs.com/package/deepagents) от LangChain): встроенное планирование задач, пользовательские инструменты и **Human-in-the-Loop (HITL)** для подтверждения чувствительных действий.

Связь с темой дня: **«9 — Лучшие практики AI-агентов, HITL, fallback»**.

## Что такое Deep Agents (простыми словами)

**Deep Agent** — это готовый «умный агент» поверх LangGraph. Вместо того чтобы вручную собирать граф из узлов и рёбер, вы вызываете одну функцию `createDeepAgent`, передаёте модель, инструменты и промпт — и получаете агента, у которого уже есть встроенные возможности, которых **нет в голом LangGraph**.

**Чем отличается от обычного LangGraph:**

- **LangGraph** — это «конструктор»: вы сами описываете граф (узлы, переходы, состояние), сами добавляете планирование, файлы, подзадачи и паузы для человека. Гибко, но всё нужно писать и склеивать самому.
- **Deep Agents** — это «агент из коробки» на базе того же LangGraph: тот же цикл «модель → инструменты → решение», но поверх уже навешаны готовые инструменты и поведение. Вы только настраиваете модель, свои инструменты и промпт — планирование, файловая система, подзадачи и HITL уже встроены и работают вместе.

То есть Deep Agent — это LangGraph-агент с заранее подключёнными возможностями, которые в чистом LangGraph пришлось бы реализовывать отдельно.

## Что такое Deep Agents (возможности)

Deep Agents — это «обвязка» (harness) поверх LangGraph с готовыми возможностями:

- **Планирование** — встроенный инструмент `write_todos` для разбиения сложных задач на шаги.
- **Файловая система** — инструменты `read_file`, `write_file`, `edit_file`, `ls`, `glob`, `grep` (в этом демо не задействованы, используются свои инструменты).
- **Подзадачи (subagents)** — делегирование работы дочерним агентам через инструмент `task` (в этом примере не используются).
- **Human-in-the-Loop** — пауза перед вызовом выбранных инструментов с решением человека: approve / reject / edit.

В этом примере показаны: **кастомные инструменты** и **HITL** для одного из них.

### Чего нет в обычном LangGraph, но есть в Deep Agents

| Возможность | В LangGraph | В Deep Agents |
|-------------|-------------|----------------|
| Планирование задач (разбить цель на шаги) | Пишете узел и инструмент сами | Встроенный инструмент `write_todos` |
| Работа с файлами (чтение, запись, поиск) | Реализуете узлы/инструменты сами | Готовые `read_file`, `write_file`, `edit_file`, `ls`, `glob`, `grep` |
| Подзадачи (отдать кусок работы другому агенту) | Строите вложенный граф и вызовы сами | Встроенный инструмент `task` и настраиваемые subagents |
| Пауза для человека перед вызовом инструмента (HITL) | Реализуете прерывания и resume в графе сами | Параметр `interruptOn` и встроенная обработка |
| Сжатие длинного диалога (суммаризация) | Пишете узел и логику сами | Встроенное middleware |
| Виртуальная файловая система (память между шагами) | Храните состояние в графе сами | Готовые бэкенды (in-memory, диск, store) |

Итого: в LangGraph вы собираете это из кубиков; в Deep Agents кубики уже собраны, вы только подключаете модель и свои инструменты.

## Ключевые особенности проекта

1. **Два инструмента**
   - `get_weather` — запрос «погоды» в городе (заглушка, без паузы).
   - `send_notification` — отправка уведомления; перед вызовом агент останавливается и ждёт подтверждения человека (HITL).

2. **Настройка HITL**
   - `interruptOn: { send_notification: true }` — перед каждым вызовом `send_notification` выполнение прерывается.
   - Используется **checkpointer** (`MemorySaver`), чтобы состояние диалога сохранялось между паузой и возобновлением.

3. **Обработка прерывания в CLI**
   - При прерывании выводятся ожидающие вызовы инструмента и аргументы.
   - Пользователь вводит решение: `approve` или `reject` (по умолчанию — approve).
   - После ввода решений агент продолжает работу через `Command({ resume: { decisions } })`.

4. **Режим без интерактива**
   - Переменная окружения `DEEP_AGENTS_AUTO_APPROVE=1` (или `true`) — все запросы на подтверждение автоматически одобряются (удобно для скриптов и CI).

5. **Модель и окружение**
   - Модель: OpenAI-совместимый API (`ChatOpenAI`), как в других примерах 9-day.
   - Переменные: `OPENAI_API_KEY`, при необходимости `OPENAI_BASE_URL`; `.env` загружается из родительской папки `node`.

## Запуск

### Требования

- Node.js 18+
- В корне `node` или в папке примера файл `.env` с `OPENAI_API_KEY=...`

### Установка

```bash
cd code/week-3/9-day/node/4-deep-agents
npm install
```

### Запуск с запросом

```bash
# Простой запрос (без HITL)
node index.js "Какая погода в Москве?"

# Запрос, при котором агент может вызвать send_notification — появится пауза для подтверждения
node index.js "Отправь уведомление Васе: встреча в 15:00"
```

### Без интерактива (автоподтверждение)

```bash
DEEP_AGENTS_AUTO_APPROVE=1 node index.js "Отправь уведомление админу: тест"
```

### Через npm-скрипты

```bash
npm run query -- "Ваш запрос"
```

## Структура кода (index.js)

- **Инструменты** — `get_weather`, `send_notification` (описание и схемы через `zod`).
- **Агент** — `createDeepAgent` с `model`, `tools`, `systemPrompt`, `interruptOn`, `checkpointer`.
- **Цикл HITL** — после `invoke` проверка `result.__interrupt__`; при наличии — вывод ожидающих действий, ввод решений пользователя (или авто-approve), затем `invoke(new Command({ resume: { decisions } }), config)`.
- **Форматирование и комментарии** — в стиле примера `3-hitl`: блоки с `=====`, комментарии на русском.
