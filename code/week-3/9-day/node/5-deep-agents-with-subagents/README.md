# Deep Agents (deepagents) — демонстрационный проект

Этот проект показывает, как собрать «deep agent» на базе пакета `deepagents` (LangChain + LangGraph):

- **LLM через Polza.ai** (OpenAI-совместимый API) и модель **`openai/gpt-5.5`**
- **tools**: кастомный инструмент `get_server_time_iso`
- **subagents**: отдельный субагент `math-assistant` со своим инструментом `multiply_numbers`, вызываемый через встроенный `task`
- **MemorySaver**: сохранение состояния диалога на уровне **thread** (мульти-тёрн)
- **Store + CompositeBackend**: маршрутизация «файлов» агента, чтобы можно было отделить «обычные» файлы от `/memories/`

Ключевые файлы:

- `index.js` — весь код демо (создание агента + запуск)
- `.env.example` — шаблон переменных окружения
- `package.json` — зависимости и команды запуска

Полезная документация:

- Deep Agents (JS): `https://docs.langchain.com/oss/javascript/deepagents/overview`
- Память LangGraph (MemorySaver, `thread_id`): `https://docs.langchain.com/oss/javascript/langgraph/add-memory`

## Требования

- **Node.js 20+** (пакет `langchain` требует Node 20)
- Пакет **`@langchain/anthropic`** установлен как зависимость, потому что `deepagents` импортирует его в сборке. Ключ Anthropic **не нужен**, если вы используете только `ChatOpenAI` + Polza.

## Установка

```bash
npm install
```

## Настройка окружения

1. Скопируйте `.env.example` → `.env`
2. Заполните ключ:

- **`POLZA_AI_API_KEY`**: ключ Polza.ai

Опционально:

- **`POLZA_MODEL`**: имя модели (по умолчанию `openai/gpt-5.5`)
- **`POLZA_BASE_URL`**: базовый URL API (по умолчанию `https://polza.ai/api/v1`)

Важно:

- **Не коммитьте** `.env` в git (там секреты).
- Рекомендуемый формат без кавычек:

```env
POLZA_AI_API_KEY=...
POLZA_MODEL=openai/gpt-5.5
POLZA_BASE_URL=https://polza.ai/api/v1
```

## Запуск

- Запустить демо-сценарий (2 последовательных вызова в одном `thread_id`, чтобы показать память):

```bash
npm start
```

- Запустить с вашим запросом (один вызов):

```bash
npm run query -- "Ваш текст"
```

## Что именно демонстрируется

### 1) Подключение LLM (Polza.ai + GPT‑5.5)

В `index.js` модель создаётся через `ChatOpenAI` из `@langchain/openai`, но с **OpenAI-совместимой** конфигурацией:

- **`configuration.baseURL`** → `POLZA_BASE_URL`
- **`apiKey`** → `POLZA_AI_API_KEY`
- **`model`** → `POLZA_MODEL` (по умолчанию `openai/gpt-5.5`)

Это позволяет использовать Polza как «шлюз», сохранив привычный OpenAI-совместимый формат.

### 2) Tools (кастомные инструменты)

В демо добавлен tool `get_server_time_iso`:

- **назначение**: вернуть серверное время в ISO 8601
- **зачем**: показать, как подключать свои tools поверх встроенных возможностей deepagents

Важный момент: deepagents добавляет и **встроенные** инструменты (например, планирование через `write_todos`, файловые операции, запуск субагентов и т.д.). В этом проекте мы специально добавляем ещё один «прикладной» tool, чтобы было видно, как расширять агента.

### 3) Subagents (субагенты и делегирование)

В конфиг агента добавлен субагент:

- **имя**: `math-assistant`
- **идея**: «карантин контекста» — узкая задача выполняется в отдельном изолированном контуре, чтобы не засорять основной контекст
- **инструменты субагента**: `multiply_numbers`

Главный агент **не имеет прямого доступа** к `multiply_numbers` — он должен делегировать выполнение через встроенный tool **`task`**, выбирая `math-assistant`.

### 4) MemorySaver (память диалога по thread_id)

Память диалога включается через `checkpointer: new MemorySaver()` и передачу `thread_id` в `invoke`:

- один и тот же **`thread_id`** → продолжение диалога, доступ к прошлым сообщениям
- другой **`thread_id`** → новая независимая сессия

В демо это видно по двум вызовам `agent.invoke(...)` с одинаковым `DEMO_THREAD_ID`.

### 5) Store + CompositeBackend (псевдо-ФС агента и «долговременная» зона)

Deep Agents предоставляет агенту виртуальную файловую систему. В этом проекте включён `CompositeBackend`, который маршрутизирует операции по префиксу пути:

- **по умолчанию**: `StateBackend` — хранение «файлов» вместе с состоянием выполнения (удобно для текущей сессии)
- **`/memories/`**: `StoreBackend` — хранение через LangGraph Store (`InMemoryStore` в демо)

Почему так сделано:

- вы можете разделить «рабочие файлы» и «зону долгой памяти» по префиксу пути
- в реальном приложении вместо `InMemoryStore` обычно подключают persistent store (БД/kv/etc.), но демо держит всё в памяти процесса

## Логика демо-сценария (если запускать без аргументов)

`npm start` делает два шага:

1. Первое сообщение: попросить умножение и время, а также сохранить имя пользователя в `/notes/user_prefs.txt`.
2. Второе сообщение: проверить, что агент «помнит» имя и числа из прошлого шага — за счёт `MemorySaver` и одинакового `thread_id`.

Если передать аргумент (`npm run query -- "..."`), выполняется один `invoke`.

## Как расширять проект под себя

- **Добавить новый tool**: создайте `tool(async (input) => ..., { name, description, schema })` и добавьте в `createDeepAgent({ tools: [...] })`.
- **Добавить субагента**: опишите объект субагента (name/description/systemPrompt/tools) и добавьте в `subagents: [...]`.
- **Сделать реальную долгую память**:
  - оставить `MemorySaver` для тредов (короткая память)
  - заменить `InMemoryStore` на persistent store (в зависимости от вашего окружения)
- **Изменить правила поведения**: отредактируйте `systemPrompt` в `index.js` — именно там задаются «политики» использования tools/subagents/файлов.

## Траблшутинг

- **Ошибка `POLZA_AI_API_KEY`**: проверьте, что `.env` существует и переменная задана, затем перезапустите процесс.
- **Неправильный `baseURL`**: убедитесь, что `POLZA_BASE_URL` ровно `https://polza.ai/api/v1` (или ваш актуальный endpoint).
- **Модель не найдена**: уточните имя модели в Polza и задайте корректный `POLZA_MODEL`.
