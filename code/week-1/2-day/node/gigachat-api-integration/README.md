# GigaChat API

Простой REST API сервер для взаимодействия с GigaChat - языковой моделью от Сбера.

## Описание

Этот проект представляет собой простой Express.js сервер, который обеспечивает взаимодействие с API GigaChat. Сервер обрабатывает аутентификацию и предоставляет единую конечную точку для отправки запросов к модели.

## Требования

- Node.js (версия 18 или выше)
- npm (входит в состав Node.js)
- Учетная запись и ключ доступа к GigaChat API

## Установка

1. Клонируйте репозиторий:
```bash
git clone [url-репозитория]
cd gigachat-api
```

2. Установите зависимости:
```bash
npm install
```

3. Создайте файл `.env` в корневой директории проекта со следующими переменными:
```env
PORT=3000
AUTH_URL=https://ngw.devices.sberbank.ru:9443/api/v2/oauth
AI_URL=https://gigachat.devices.sberbank.ru/api/v1/chat/completions
AUTH_KEY_PERS=ваш_ключ_авторизации
```

## Запуск

Для запуска сервера выполните:
```bash
npm start
```

Сервер запустится на порту, указанном в `.env` файле (по умолчанию 3000).

## Использование

### Отправка запроса к GigaChat

**Endpoint:** `POST /ai`

**Тело запроса:**
```json
{
    "text": "Ваш вопрос или промпт"
}
```

**Пример запроса с использованием curl:**
```bash
curl -X POST http://localhost:3000/ai \
     -H "Content-Type: application/json" \
     -d '{"text": "Привет, как дела?"}'
```

## Структура проекта

- `server.js` - основной файл сервера
- `auth.js` - модуль аутентификации
- `.env` - файл с переменными окружения (необходимо создать)
- `package.json` - файл с зависимостями и скриптами

## Зависимости

- express - веб-фреймворк
- axios - HTTP-клиент
- dotenv - для работы с переменными окружения
- cors - для настройки CORS

## Безопасность

- Убедитесь, что файл `.env` добавлен в `.gitignore`
- Не передавайте ключ авторизации в открытом виде
- Используйте HTTPS при развертывании в продакшн
