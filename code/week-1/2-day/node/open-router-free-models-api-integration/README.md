# DeepSeek API Integration

Простой REST API сервер для взаимодействия с DeepSeek - языковой моделью через OpenRouter API.

## Описание

Этот проект представляет собой простой Express.js сервер, который обеспечивает взаимодействие с DeepSeek API через OpenRouter. Сервер предоставляет единую конечную точку для отправки запросов к модели DeepSeek Chat v3.1.

## Требования

- Node.js (версия 18 или выше)
- npm (входит в состав Node.js)
- API ключ от OpenRouter для доступа к DeepSeek

## Установка

1. Клонируйте репозиторий:
```bash
git clone [url-репозитория]
cd deepseek-api-integration
```

2. Установите зависимости:
```bash
npm install
```

3. Создайте файл `.env` в корневой директории проекта со следующими переменными:
```env
PORT=3000
AI_URL=https://openrouter.ai/api/v1/chat/completions
OPENROUTER_API_KEY_DEEPSEEK=ваш_api_ключ_от_openrouter
```

## Запуск

Для запуска сервера выполните:
```bash
npm start
```

Сервер запустится на порту, указанном в `.env` файле (по умолчанию 3000).

## Использование

### Отправка запроса к DeepSeek

**Endpoint:** `POST /ai`

**Тело запроса:**
```json
{
    "message": "Ваш вопрос или промпт"
}
```

**Пример запроса с использованием curl:**
```bash
curl -X POST http://localhost:3000/ai \
     -H "Content-Type: application/json" \
     -d '{"message": "Привет, как дела?"}'
```

**Пример ответа:**
```json
"Привет! У меня всё хорошо, спасибо за вопрос. Как дела у вас?"
```

## Структура проекта

- `server.js` - основной файл сервера
- `.env` - файл с переменными окружения (необходимо создать)
- `package.json` - файл с зависимостями и скриптами

## Зависимости

- express - веб-фреймворк
- axios - HTTP-клиент для запросов к OpenRouter API
- dotenv - для работы с переменными окружения

## Получение API ключа

1. Зарегистрируйтесь на [OpenRouter](https://openrouter.ai/)
2. Получите API ключ в разделе "Keys"
3. Добавьте ключ в файл `.env` как `OPENROUTER_API_KEY_DEEPSEEK`

## Безопасность

- Убедитесь, что файл `.env` добавлен в `.gitignore`
- Не передавайте API ключ в открытом виде
- Используйте HTTPS при развертывании в продакшн
