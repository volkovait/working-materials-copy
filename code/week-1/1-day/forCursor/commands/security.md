# Команда: /security

## Описание
Проверь код на уязвимости безопасности.

## Prompt
Проведи security audit этого кода:

1. **Injection атаки:**
   - ✅ SQL Injection: используются ли prepared statements?
   - ✅ NoSQL Injection: санитизация в MongoDB запросах?
   - ✅ Command Injection: проверка shell команд?
   - ✅ XSS: экранирование пользовательского ввода?

2. **Аутентификация и авторизация:**
   - ✅ Проверка прав доступа перед операциями
   - ✅ JWT токены валидируются правильно?
   - ✅ Session management безопасен?
   - ✅ Password hashing (bcrypt, argon2)?

3. **Работа с данными:**
   - ✅ Валидация всех входных данных
   - ✅ Секреты не хардкодятся в коде
   - ✅ Чувствительные данные не логируются
   - ✅ HTTPS для передачи данных?

4. **CSRF защита:**
   - ✅ CSRF токены для state-changing операций
   - ✅ SameSite cookies
   - ✅ Проверка origin/referer

5. **Rate Limiting:**
   - ✅ Ограничение частоты запросов
   - ✅ Защита от brute force атак
   - ✅ Throttling для дорогих операций

6. **Зависимости:**
   - ✅ Актуальные версии библиотек
   - ✅ Известные уязвимости (npm audit)

**Результат:**
Список найденных уязвимостей с уровнем критичности и примеры исправлений:
- 🔴 **CRITICAL:** Требует немедленного исправления
- 🟠 **HIGH:** Исправить в ближайшее время
- 🟡 **MEDIUM:** Желательно исправить
- 🟢 **LOW:** Улучшение безопасности
