# Команда: /tests

## Описание
Создай comprehensive тесты для выбранного кода.

## Prompt
Создай тесты для этого кода:

1. **Unit тесты:**
   - Проверка основной функциональности
   - Граничные случаи (empty, null, undefined)
   - Edge cases (минимум, максимум, границы)

2. **Негативные тесты:**
   - Некорректные входные данные
   - Обработка ошибок
   - Исключительные ситуации

3. **Integration тесты (если применимо):**
   - Взаимодействие с другими модулями
   - API запросы
   - Database операции

4. **Требования:**
   - Используй современный фреймворк (Jest/Vitest для JS/TS, Pytest для Python)
   - Добавь описательные названия тестов
   - Используй моки для внешних зависимостей
   - Стремись к 100% покрытию критичных путей

**Структура:**
```
describe('название модуля', () => {
  describe('метод/функция', () => {
    it('должен делать X когда Y', () => {
      // arrange, act, assert
    });
  });
});
```
