## 🧩 Элементы хорошего промпта

Качественный промпт состоит из нескольких понятных частей. Ниже — краткая структура с примерами, которые можно сразу применять.

### 📝 Инструкция (что нужно сделать)

- **Сформулируйте одну главную задачу** в повелительном наклонении
- **Уточните ограничения**: стиль, тон, длина, критерии качества
- **Опишите, чего делать не нужно**, если это критично
- При необходимости **задайте роль модели**

Пример:
```text
Ты — редактор деловой аналитики. Суммируй данный текст в 3–4 предложения
для занятой аудитории руководителей. Избегай жаргона. Тон — нейтральный.
```

### 📚 Контекст (что модели известно заранее)

- Дайте фон: аудитория, цель, доменная область
- Приложите справку, правила, глоссарий или формат
- Отделяйте контекст от входных данных визуально

Пример:
```text
Контекст:
- Компания: B2B SaaS в финтехе
- Аудитория: продуктовые менеджеры и руководители отделов
- Цель: быстро понять суть и принять решение
```

### 🔢 Входные данные (что пользователь вводит)

- Ясно отделяйте вход с помощью разделителей
- Указывайте ожидаемый тип данных (текст, список, JSON)

Пример:
```text
Входные данные (между тройными кавычками):
"""
<сюда вставьте исходный текст>
"""
```

### 📤 Индикатор вывода (формат и критерии результата)

- Заранее укажите формат: текст, список, таблица, JSON
- Добавьте критерии: длина, стиль, поля, сортировка

Примеры:
```text
Формат вывода: маркированный список из 5 пунктов, каждый — не длиннее 20 слов.
```

```text
Формат вывода: валидный JSON без комментариев со следующими полями:
{
  "title": string,
  "summary": string,
  "key_points": string[]
}
```

### 🧪 Цельный шаблон (всё вместе)

```text
Ты — редактор деловой аналитики.

Инструкция:
Суммируй входной текст в 3–4 предложения для занятой руководящей аудитории.
Избегай жаргона и оценочных суждений. Тон — нейтральный.

Контекст:
- Компания: B2B SaaS в финтехе
- Цель: быстро понять суть и принять решение

Входные данные (между тройными кавычками):
"""
<сюда вставьте исходный текст>
"""

Формат вывода: валидный JSON без комментариев:
{
  "title": "краткий заголовок",
  "summary": "3–4 предложения",
  "key_points": ["пункт 1", "пункт 2", "пункт 3"]
}
```

### ✅ Краткая памятка

- **Инструкция**: что именно сделать и зачем
- **Контекст**: что уже известно модели
- **Входные данные**: что подставляет пользователь
- **Индикатор вывода**: в каком виде вернуть результат
