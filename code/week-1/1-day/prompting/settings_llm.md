### 🎛️ Настройки LLM: как управлять ответами модели

При работе с промптами вы взаимодействуете с LLM через API или напрямую. Ниже — краткие объяснения ключевых параметров и практические советы по их применению.

### 🌡️ Температура (temperature)

Чем ниже температура, тем более детерминированным будет ответ (модель выбирает самый вероятный следующий токен). Повышение температуры увеличивает случайность и творческость.

- Рекомендуется: для фактических, точных ответов используйте низкие значения (0–0.3). Для креативных задач — более высокие (0.7–1.0).
- Пример промпта (низкая температура): «Кратко и точно объясни, что такое квантовая запутанность.»
- Пример промпта (высокая температура): «Придумай 5 необычных идей для экологичного стартапа в сфере энергии.»

### 🎯 Ядровое сэмплирование (top_p)

`top_p` — альтернатива температуре: модель выбирает токены из наименьшего возможного набора, чья суммарная вероятность ≥ `p` (сэмплирование ядра). Низкий `top_p` делает ответы более предсказуемыми, высокий — разнообразными.

- Общая рекомендация — изменять либо `temperature`, либо `top_p`, но не оба одновременно.
- Пример промпта (низкий top_p): «Дай точное определение термина “энтропия” одним предложением.»
- Пример промпта (высокий top_p): «Сгенерируй 3 альтернативных слогана для городского велошеринга.»

### 📏 Максимальная длина ответа (max_tokens)

Ограничивает количество генерируемых токенов. Полезно для контроля затрат и предотвращения слишком длинных ответов.

- Рекомендуется: задавайте явный лимит для коротких ответов, резюме и списков.
- Пример промпта: «Сформулируй резюме текста в 2–3 предложениях.»

### ⛔ Стоп‑последовательности (stop_sequences)

Строки, при встрече с которыми модель прекращает генерацию. Позволяют жёстко ограничивать структуру ответа.

- Пример: «Составь нумерованный список пунктов с 1 по 10» и указать стоп‑последовательность «11», чтобы остановить генерацию после пункта 10.

### 🔁 Штраф за частоту (frequency_penalty)

Накладывает штраф на токены пропорционально тому, как часто они уже встречались в подсказке/ответе. Чем выше значение, тем меньше повторов слов и фраз.

- Рекомендуется: повышать для борьбы с речевыми повторами в длинных текстах.
- Пример промпта: «Напиши обзор книги без повторов ключевых слов, избегая тавтологии.»

### 🧭 Штраф за наличие (presence_penalty)

Применяет одинаковый штраф ко всем повторяющимся токенам, вне зависимости от частоты. Повышение `presence_penalty` стимулирует введение новых тем и лексики.

- Рекомендуется: увеличивать для более разнообразного, «исследовательского» текста; снижать — для фокуса на теме.
- Пример промпта: «Предложи разные подходы к обучению математике для подростков, избегая однообразных формулировок.»

### ✅ Практические рекомендации

- Меняйте один параметр за раз: либо `temperature` или `top_p`; либо `frequency_penalty` или `presence_penalty`.
- Для фактов и точных ответов: `temperature` 0–0.3, `top_p` 0.8–1.0, штрафы 0.
- Для креатива и брейншторма: `temperature` 0.7–1.0, `top_p` 0.9–1.0, `presence_penalty` 0.3–0.6.
- Контролируйте форму ответа через `max_tokens` и `stop_sequences` (особенно для списков и коротких форм).
- Фиксируйте параметры для воспроизводимости результатов и последующей отладки промптов.
При работе с промптами вы взаимодействуете с LLM с помощью API или непосредственно. Вы можете настроить несколько параметров, чтобы получить различные результаты для ваших промптов.
