# Exam: Веселая ферма

## Введение

Необходимо создать `MVP` приложение для виртуальных фермеров. Пользователи твоего приложения смогут добавлять на свою ферму зверушек, а также посещать соседние фермы чтобы смотреть, что на них есть.

**Сначала прочитай всё задание целиком!**
Если не получается сделать какой-то релиз, переходи к следующему.

## Pre-release: Setup

Проинициализируй проект, продумай скетч приложения, создай базу данных, накати сиды и пропиши переменные окружения. 

## Release 0: 

Создай все маршруты на сервере. 
Создай все необходимые компоненты на клиенте. 

Приложение может выглядеть следующим образом:
- `"/"` — главная страница сайта, отображает все созданные фермы. Для гостей приложения отобрази кнопку `Создай свою ферму` с переходом на страницу создания фермы.
![no user main animation](readme-assets/p1.png)  
*Рисунок 1*. no user mainPage.

- `"/farm/create"` — отображается форма для РЕГИСТРАЦИИ фермы с именем владельца, уникальным email и паролем. Под каждого пользователя заводится ровно одна ферма, которая привязана к его аккаунту.
- `"/login"` — отображается форма для аутентификации (`email`, `password`)
- `"/farm/:id"` — показывает ферму по `id` пользователя (если пользователь зарегистрирован и находится на своей странице, то он может добавить нового питомца или удалить старого. Если пользователь не авторизован или находится на чужой странице может посмотреть питомцев, но не видит форму добавления и кнопку удаления)

## Release 1: Login/Logout

Добавь в шапку сайта кнопки `Login` и `Create Farm`, ведущие на соответствующие страницы. Не забудь обрабатывать ошибки: если у пользователя не получается залогиниться - ему нужно знать причину. При успешном входе - должна появиться кнопка `Logout`. При нажатии на `Logout` пользователь выходит из системы и оказывается на домашней странице. На данном этапе домашняя страница может быть пустой, позже на ней будут отображаться ссылки на фермы других пользователей.

## Release 2: UI

После того, как все страницы открываются по своим URL, подумай, каким образом можно разбить интерфейс на компоненты. Вот некоторые примеры:
1) `NavBar` — навигационная панель отображается на всех страницах. Сделай так, чтобы слева отображалось `Ферма Alex` (если name Alex). Кнопки навигации должны быть разными у гостя и авторизованного пользователя.
2) `MainPage` — содержит в себе кнопки перехода на фермы зарегистрированных пользователей. Если пользователь является гостем -- отобрази кнопку перехода на страницу с добавлением фермы.
3) `UserFarmPage` — страница фермы по `id`. На ней должны отображаться все животные фермера.
4) `AnimalCard` — карточка с отображением одного животного на ферме

Продумай все UI-компоненты, которые тебе понадобятся. Создай их и подключи в приложение.

## Release 3: CRUD для питомцев

Пропиши логику для добавления животных на свою ферму -- используй форму на персональной странице фермы. На фермах животные не только появляются, но и исчезают 🐄🔪🥩 -- реализуй логику удаления. 
Добавление и удаление должны происходить через AJAX на одной и той же странице фермы. Обрати внимание на маршруты и сделай их согласно тестам.

Важно, чтобы элементы взаимодействия (форма добавления и кнопки) были скрыты для пользователя, который не является владельцем фермы.

Добавь пользователю возможность менять имя и картинку своим питомцам его фермы любым удобным способом.


## Release 4: UX и обработка ошибок

Защитись от уязвимостей:

- при обращении по пути `"/farm/:id"` если передан некорректный `id` приложение не должно упасть.
- неавторизованный гость не должен иметь возможности ни добавлять, ни удалять питомцев. Даже `fetch` запросом из браузера
- проверь, что приложение запускается и может корректно работать при отстутствии авторизации
- при обращении по несуществующему пути `"/ololo/trololo"` приложение должно не упасть, а показать ошибку отсутствия страницы

## Release 5: Милые животные

Видишь -- наши животные совсем уже заскучали. Давай оживим ферму! Пусть каждое животное можно погладить (добавь для этого кнопку). А если животное погладили 10 раз, его карточка должна как-то визуально измениться. Например, сверху карточки должен появиться счастливый эмоджи 🥰 

Статус довольного животного должен сохраняться при обновлении страницы. Пусть погладить 10 раз можно любым пользователям, даже если это кто-то один любит всех гладить. Нам главное, чтобы всем дарили счастье!

## Release 6*: Трейдинг

Благодаря такому замечательному приложению наши животные превращаются в самые настоящие NFT! Для полноценной миграции нам потребуется система передачи прав между разными пользователями. 

Если пользователь является владельцем конкретного животного, отобрази кнопку `"подарить"`. При нажатии на эту кнопку должно всплывать модальное окно с вопросом 'Кому подарить?' и инпутом. При вводе текста в инпут, должен происходить поиск пользователя с email, пересекающимся с введённым текстом. Нужный email можно выбрать и отправить животное своему другу. После передачи животное должно пропадать с фермы отправителя и появляться на ферме получателя.
