# Техническое задание для проекта "Новостной сайт"

## Описание проекта

Необходимо разработать новостное приложение, где пользователи могут просматривать новости, а зарегистрированные авторы — создавать и редактировать их. Некоторые новости могут содержать контент, неприемлемый для просмотра на рабочем месте (NSFW): такие новости будут доступны только зарегистрированным пользователям после подтверждения согласия на просмотр.

### Авторы:

- Для размещения статей авторам необходимо зарегистрироваться.
- Поля автора: имя, email, пароль.
- Регистрация и авторизация могут быть реализованы на отдельных страницах или через модальное окно.
- После успешной регистрации или авторизации в интерфейсе отображается имя пользователя.

### Статьи:

- Статьи содержат следующие поля: заголовок, текст статьи, изображение, пометку о наличии контента, неприемлемого на рабочем месте (NSFW).
- Картинки можно добавлять либо по ссылке на внешний ресурс, либо загружать на сервер.
- При отображении статьи должно быть видно имя автора.
- Статьи с пометкой о неприемлемом контенте недоступны для просмотра незарегистрированным пользователям.

### Страницы:

Приложение должно содержать следующие страницы:

- Главная страница с заголовками пяти последних новостей.
- Страницы регистрации и входа в аккаунт.
- Страница или модальное окно для добавления новой новости.
- Страница отдельной новости.
- Личный кабинет автора со списком его новостей.

## Релизы:

### Release 0:

Продумай архитектуру базы данных и набросай структуру страниц приложения. Создай модели и миграции для работы с данными, а также заготовки компонентов пользовательского интерфейса.

### Release 1:

Добавь кнопки входа и регистрации в навигационную панель. Реализуй логику этих страниц. В случае ошибки авторизации пользователю должно отображаться соответствующее сообщение. После входа в систему в навигации отображается имя пользователя.

### Release 2:

Реализуй функционал добавления новостей для авторизованных пользователей. Этот функционал должен быть недоступен гостям (неавторизованным пользователям). При добавлении новости предусмотрена возможность пометить её как содержащую контент NSFW.

### Release 3:

Отобрази заголовки пяти последних новостей на главной странице, указывая имя автора рядом с каждой новостью. Полное содержание статьи (включая изображение и текст) отображается на отдельной странице.

Для новостей с неприемлемым контентом (NSFW) необходимо обеспечить возможность просмотра только после выполнения следующих условий:

1. Пользователь авторизован на сайте.
2. Пользователь подтвердил согласие на просмотр.

Подтверждение может быть реализовано через всплывающее окно, чекбокс, модальное окно или другой способ на твой выбор.

### Release 4:

Реализуй личный кабинет, где авторы могут видеть список своих новостей (заголовки без текста и изображений). Рядом с каждой новостью должна быть кнопка удаления, которая видна только в личном кабинете. Также добавь возможность редактировать свои новости удобным способом.

### Release 5:

При большом количестве новостей добавь возможность поиска по тексту. Оптимизируй поиск с помощью механизмов, таких как `debounce` или `throttle`, чтобы уменьшить нагрузку на сервер.

### Release 6:

Добавь страницу профиля автора, на которой пользователи (включая гостей) смогут увидеть список всех новостей, опубликованных данным автором.
