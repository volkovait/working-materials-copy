# Phase 2 assessment: AlpineCode Recruit

## Введение

Необходимо разработать приложение для подбора персонала. HR-сотрудники, использующие твоё приложение, должны иметь возможность взаимодействовать с кандидатами и передавать их друг другу. Твоё приложение поможет упростить процесс подбора персонала и улучшить его эффективность.

**Сначала прочитай всё задание целиком!**

Если не получается сделать какой-то релиз, переходи к следующему. Делай коммиты почаще, но не выполняй `git push` раньше времени.

## Pre-Release

Подготовь структуру проекта и базы данных. В начале тебе потребуются две модели: "Пользователи" и "Кандидаты". "Пользователи" должны содержать необходимую информацию для аутентификации, а "Кандидаты" - имя, фото, пол, возраст и поле для заметок, которое можно использовать для описания или комментариев.

Выполни начальную инициализацию, накати миграции и подготовь сиды. Пропиши переменные окружения.

## Release 0: Pages

Подготовь нужные страницы, которые будут в твоём приложении. Минимальный набор должен включать:
- Страницу авторизации
- Личный кабинет пользователя с кандидатами
- Личную страницу кандидата
- Страницу с принятыми на работу кандидатами

## Release 1: Authentication

Разработай логику аутентификации пользователя. Гарантируй, что личный кабинет доступен только авторизованным пользователям. Добавь кнопку для выхода из учетной записи, которая не видна гостям на сайте.

## Release 2: Candidates

В личном кабинете пользователя реализуй функционал добавления новых кандидатов. При необходимости, предоставь отдельную страницу для этой цели. Все кандидаты, добавленные рекрутерами, должны быть связаны с ними.

Создай карточку кандидата. На каждой карточке будет только имя и фото кандидата, а также должны быть кнопки:
- Принять (на работу)
- Удалить
- Подробнее (переход на страницу кандидата)

## Release 3: Candidates CRUD

Реализуй логику удаления кандидата с помощью кнопки на странице личного кабинета. Подготовь страницу с подробной информацией о кандидате и реализуй функционал редактирования с использованием удобных методов.

Добавь к личной странице кандидата большое поле для заметок.

## Release 4: Approval

Реализуй функционал кнопки "Принять на работу". По клику на кнопку:
- Должна появиться метка о том, что кандидат успешно принят
- Кнопка "Принять на работу" должна скрыться
- Вместо нее должна появиться кнопка "Уволить"

Все кандидаты, принятые на работу рекрутерами, должны отображаться на странице принятых кандидатов. Кнопки для взаимодействия должны быть доступны только рекрутерам, которые приняли кандидата на работу.

## Release 5: Dashboard

В личном кабинете рекрутера добавь разные статистические данные, такие как:
- Общее количество добавленных рекрутером кандидатов
- Количество удаленных кандидатов
- Количество принятых на работу кандидатов
- Средний возраст кандидатов

Ты можешь добавить другие метрики по своему усмотрению. Сравни производительность рекрутеров, например, показав топ-3 самых продуктивных.


## Release 6*: Transfer

Если кандидат не подходит одному из рекрутеров, но может быть полезным для другой команды, реализуй функционал передачи кандидата от одного рекрутера другому. При получении нового кандидата, пользователь должен получить уведомление о передаче, используя протокол WebSocket.
