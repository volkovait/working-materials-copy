# Эльбрус-Лотерея

## Введение

Тебе предстоит создать свою собственную платформу для лотерейных билетов, где пользователи смогут попытать удачу и, быть может, выиграть что-то очень ценное!

## Процесс

Конец экзамена в 16:00, во время обеда с 12:30 до 14:00 код не пишем. Не забывай почаще делать коммиты. Пулл-реквест нужно оставлять только после 15:50; лучше не откладывать это до последней минуты.

## Release 0

Вначале прочитай задание целиком и ознакомься с ТЗ. Проинициализируй проект и подготовь базу данных. Создай 2 модели:

- `User`
  - email
  - пароль (захэшированный)
  - имя пользователя
  - поле `admin` с булевым значением
- `Ticket`
  - номер билета в виде строки
  - ссылка на id владельца (разреши значение `null` и записывай его по умолчанию)
  - сумма приза в виде числа

Сразу подготовь сиды на пользователя, у которого в поле `admin` будет записано `true`. В дальнейшем в приложении никто из юзеров не будет админом. Кстати, будет удобно, если ты подготовишь сиды и для парочки лотерейных билетов, у которых изначально не будет владельца.

Создай сервер и подними приложение.

## Release 1

Создай страницы и ссылки в навбаре:

- `'/'`, Главная страница с описанием лотереи и списком свободных билетов
- `'/login'`, страница с формой входа
- `'/signup'`, страница с формой регистрации
- `'/tickets'`, страница личного кабинета со списком билетов пользователя
- `'/admin'`, страница для администратора

Пропиши логику защиты страниц и скрытия кнопок в навбаре:

- Главная страница доступна всем
- Страница авторизации и регистрации доступна только неавторизованным пользователям
- Личный кабинет доступен авторизованным пользователям
- Страница `'/admin'` доступна только для авторизованного пользователя, который является администратором

Не забудь добавить кнопку для выхода из учётной записи.

## Release 2

Давай наполним страницы контентом.

### Auth

Добавь формы и реализуй логику авторизации и регистрации. Не забудь при добавлении нового пользователя в базу данных прописывать в поле `admin` значение `false`.

### Главная

Создай компонент карточки одного лотерейного билета. Выведи все билеты, у которых нет владельца, на главной странице. Сделай так, чтобы полный номер лотерейного билета был недоступен для просмотра.

## Release 3

Пропишем логику личного кабинета. На страничке `'/tickets'` должны отображаться билеты авторизованного пользователя. Но вначале эти билеты нужно как-то заполучить!

На эту страничку добавь форму, куда пользователь может ввести номер существующего билетика и стать его владельцем. Если такой свободный билет есть, добавь его к существующим на данной странице.

## Release 4

Давай в личном кабинете участника добавим функционал удаления лотерейного билета. Сделай кнопку и защити эндпоинт.

## Release 5

Пусть теперь администратор сможет удалять и редактировать билеты на главной странице. Реализуй это любым удобным тебе способом.

## Release 6

Теперь давай пропишем валидации и обработаем потенциальные ошибки:

- Когда юзер хочет добавить несуществующий билет, ему нужно выдать сообщение, что такого билета нет
- Когда юзер хочет добавить к себе билет, у которого уже есть другой владелец, выведи сообщение, что билет уже занят
- Когда админ хочет создать новый билет с номером, который уже используется, выведи сообщение об ошибке
- Защити эндпоинты, чтобы удалять/редактировать свободные билеты не смог никто, кроме администратора

## Release 7\*

Что если мы купили лотерейный билет, но хотим его подарить другу? В личном кабинете добавь в поиске возможность присвоить новый билет другому зарегистрированному пользователю. Удобно, если выпадет dropdown с именами или email-ами.

А ещё, чтобы мотивировать людей покупать больше лотерейных билетов, давай в личном кабинете отобразим сумму всех призовых денег по всем билетам пользователя и напишем что-то вроде "Ты можешь выиграть `n` рублей!"
