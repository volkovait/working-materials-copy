# Квиз плиз

## Введение

Любишь решать задания, отвечать на вопросы и зарабатывать очки? Сегодня тебе предстоит создать именно такой уникальный сервис `Quiz pliz`. Пользователи смогут добавлять свои вопросы, указывать за них вознаграждение в баллах, и отвечать на чужие вопросы.

### Важно

Конец экзамена в 15:00, во время обеда с 12:30 до 14:00 код не пишем. Не забывай почаще делать коммиты. Пулл-реквест нужно оставлять только после 14:50; лучше не откладывать это до последней минуты. Делай `git push` только один раз — не раньше, чем за полчаса до окончания экзамена.
В течение экзамена ничего пушить не нужно!

### Рекомендации

1. Прочитай всё задание целиком.
2. Сфокусируйся на функционале приложения, оценивать будем в первую очередь его.

   Красивая вёрстка — это приятно, но может затратить слишком много времени.

3. Сделай навигацию по сайту — пользователь должен иметь возможность добраться до любой страницы через элементы на страницах — ссылки, формы, etc.

4. Если не получается сделать какой-то релиз, переходи к следующему.

## Pre-release

Проинициализируй проект, проверь самые распространённые ошибки:

- репозиторий был вначале форкнут, а потом склонирован именно форк
- всё задание прочитано
- присутствует `.gitignore`
- база данных спроектирована корректно
  - модель пользователя должна иметь данные для авторизации
  - модель вопроса должна иметь текст с описанием, правильный ответ, количество очков (вознаграждение за верно данный ответ)
  - пользователи и вопросы должны быть связаны связью One-To-Many
- подготовлены сиды
- подготовлен файл `.env` и пример для его заполнения

Обязательно добавь скрипт для быстрого запуска твоего приложения в `package.json`, например что-то такое: `"quick-start": "npx sequelize-cli db:drop && npx sequelize-cli db:create && npx sequelize-cli db:migrate && npx sequelize-cli db:seed:all && webpack -d eval-source-map && babel-node src/server.js"`

## Release 0. Структура страниц

Создай страницы и ссылки в навигационной панели:

- `/` Главная страница с описанием приложения
- `/login` Страница входа
- `/signup` Страница регистрации
- `/account` Страница личного кабинета со списком всех своих добавленных вопросов
- `/quiz` Страница с отображением вопросов, на которые можно ответить

Пропиши логику защиты страниц и скрытия кнопок в навбаре:

- Главная страница доступна всем
- Страница авторизации и регистрации доступна только неавторизованным пользователям
- Личный кабинет и страница `/quiz` доступны авторизованным пользователям

Реализуй полностью авторизацию и регистрацию. Не забудь добавить кнопку для выхода из учётной записи.

## Release 2. Редактирование

Добавь в карточке вопроса кнопку "Редактировать". По клику на неё нас должно перевести на страницу `/question/:id/edit` с формой редактирования вопроса. Поля в форме должны быть предзаполнены текущими данными. После отправки формы пользователя должно перенаправить обратно в личный кабинет.

## Release 3. Quiz

На странице `/quiz` должны отображаться 3 случайных вопроса (созданные другими авторами) и текущий счёт (изначально 0 очков). У каждого вопроса должен быть свой инпут и своя кнопка отправки, по клику на которою кнопка и инпут становятся неактивными. Если ответ был дал верный, то текущий счёт должен увеличиться, а если неверный -- остаться неизменным.

Данное задание не подразумевает долгосрочное хранение счёта или собственных ответов в базе данных, поэтому при перезагрузке страницы счётчик сбрасывается, а на странице появляются 3 новых случайных вопроса.

P.S. для генерации случайных 3 вопросов используй `order: Sequelize.literal("RANDOM()"), limit: 3`
P.P.S. На данной страничке у карточек вопросов не должно быть кнопок "Удалить, Редактировать"

## Release 4. Сохранение счёта

Всё уже реализовано? Тогда давай сохранять текущий счёт в `localStorage`, чтобы даже после перезагрузки страницы мы его не потеряли.

## Release 5. Мы храним куки

Функционал готов? Стилизуй компоненты, приведи приложение в единый вид. Добавь снизу сообщение, которое будет показываться при входе в приложение с текстом на подобии: "Мы используем файлы cookie для персонализации содержимого и объявлений, а также анализа трафика. Нажимая кнопку «Принять», вы выражаете свое согласие с этим утверждением." Сделай кнопку "Принять", при клике на которую данное сообщение скрывается. Сохрани данный выбор в сессии пользователя, чтобы в будущем не показывать это модальное окно при каждом переходе на новую страницу.
