# Экзамен: Виртуальная ферма

Задача этого экзамена - создать MVP приложения для виртуальных фермеров. Ваши пользователи смогут создавать свои фермы и добавлять на них забавных питомцев. Кроме того, они смогут посещать фермы других пользователей, чтобы посмотреть, какие питомцы у них есть.

## Шаги выполнения

1. Сделать fork репозитория с экзаменом
2. В созданном личном репозитории применяем git clone
3. Работаем и пишем код, после написания фиксируем через git commit
4. Делать постоянно git push нет необходимости, лучше его сделать в конце экзамена
5. В своём личном репозитории создаём Pull Prequest где в качестве предлагаемого кода будет ветка с вашей работой, запрос на слияние должен быть в репозиторий Elbrus

## Подготовка к работе

**Прежде всего, внимательно прочитайте все задание целиком!** Если не удается выполнить какой-то из пунктов, переходите к следующему.

Убедитесь, что в вашем проекте присутствуют все необходимые файлы, такие как .gitignore, package.json, и что установлены все необходимые библиотеки. Проинициализируйте eslint. Создайте базу данных, подготовьте данные для заполнения (сиды) и установите переменные окружения.

## Release 0: Маршрутизация

Первым шагом является создание всех маршрутов на бэкенде. Убедитесь, что они открывают нужные страницы.

В вашем приложении должны быть следующие страницы:

- Главная страница, на которой отображаются все фермы. Если пользователь авторизован, то на главной странице также должны отображаться его собственные питомцы (My pets).
- Страницы для входа и регистрации пользователей.
- Персональная страница фермы, на которой можно просматривать питомцев конкретной фермы.
- Личный кабинет пользователя с его питомцами.
- Страница с формой для добавления питомца в свою ферму.

## Release 1: Вход, Регистрация и Выход

Добавьте кнопку в шапке сайта, которая перенаправляет на страницу аутентификации. Обработайте ошибки: если пользователю не удается войти, предоставьте информацию о причине. При успешном входе кнопка входа/регистрации должна заменяться на кнопку выхода из учетной записи. При нажатии на кнопку выхода, пользователь должен выходить из системы и перенаправляться на домашнюю страницу. На данном этапе домашняя страница может быть пустой, впоследствии на ней будут отображаться ссылки на фермы других пользователей и их питомцы.

## Release 2: Интерфейс и Логика

Подготовьте все необходимые компоненты, такие как навигацию, карточки животных с кнопками, боковую панель с фермами и форму для добавления животных.

Убедитесь, что животные и фермы из сидов отображаются в соответствии с ТЗ. Реализуйте логику добавления животных в свою ферму, предоставив отдельную страницу для этой цели.

## Release 3: Редактирование и Удаление

Добавьте возможность для пользователей изменять имя и изображение своих питомцев, а также удалять тех питомцев, которые больше не проживают на их ферме. Реализуйте это любым удобным способом.

## Release 4: Улучшение качества жизни

Существует множество способов улучшить приложение. Вот что можно сделать:

- Убедитесь, что на страницах чужих ферм не отображаются кнопки для взаимодействия с животными (редактирование и удаление).
- Реализуйте загрузку изображений через `multer` вместо URL.
- Защитите эндпоинты от несанкционированного доступа, например:
  - Только авторизованный пользователь может добавить животное на ферму.
  - Только владелец животного может изменить его или удалить.
  - Авторизованный пользователь не должен иметь возможность повторно войти или зарегистрироваться.
- Обработайте ошибку, когда кто-то пытается перейти на страницу несуществующей фермы; например, подготовьте стандартную страницу `404 Not Found`.
- При выходе из аккаунта или удалении животного, предоставьте модальное окно для подтверждения выбора.

## Release 5: Уведомления

Эти животные такие милые и вызывают много положительных чувств - почему бы не поделиться ими с автором! Давайте это сделаем: на карточке животного добавьте возможность выбора одной из нескольких эмоций, например:

- :heart_eyes:
- :smile:
- :cry:
- :alien:
- :heart:

При выборе одной из этих эмоций автору питомца должно сразу же отображаться уведомление о том, что такой-то пользователь выбрал такую-то эмоцию для такого-то питомца. Реализуйте это с использованием WebSocket.
