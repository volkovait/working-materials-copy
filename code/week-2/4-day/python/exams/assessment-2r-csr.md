# Загрузка резюме

## Введение

Проект для создания, редактирования и просмотра резюме. Пользователи могут загружать свои резюме, управлять их видимостью (публичные или приватные), а также просматривать резюме других пользователей.

## Pre-release: Setup

В проекте уже есть папки `client` и `server`: перейди в них и установи требуемые зависимости. Подготовь все необходимые конфигурационные файлы, установи требуемые пакеты. Создай базу данных. Не удаляй файл с тестами `server/spec/app.spec.js`

## Releaase 0: Страницы и компоненты

Подготовь все требуемые страницы на клиенте. В приложении должны быть:
- страница входа и регистрации нового аккаунта
- страница профиля
- страница просмотра всех открытых резюме
- страница для создания резюме

Создай элементы интерфейса: кнопки, формы, навигацию

## Release 1: Регистрация и аутентификация пользователей

Создай систему регистрации пользователей и входа в существующие аккаунты. Аутентификация должна работать через email и пароль. Реализуй валидацию данных при регистрации и входе (например, email должен быть уникальным).

## Release 2: Создание резюме

Реализуй функционал создания нового резюме. Пользователь должен иметь возможность создать новое резюме, заполнив необходимые поля (образование, опыт работы, навыки и т.д.). Каждое резюме может быть публичным или приватным: публичные резюме должны быть доступны для просмотра другим пользователям, а приватные резюме доступны только тому, кто их создал.

Отобрази все публичные резюме на соответствующей странице. Пользователь должен иметь возможность просматривать публичные резюме других пользователей. Для отображения полной информации по резюме воспользуйся либо отдельной страницей, либо модальным окном.

## Release 3: Редактирование и удаление

Напиши функционал редактирования резюме любым удобным способом. Обязательно реализуй функционал изменения видимости резюме -- если пользователь публичное резюме решит скрыть, то оно должно будет пропасть для других пользователей.

Теперь сделай так, чтобы резюме можно было удалять со страницы профиля. В личном кабинете отобрази созданные пользователем резюме чтобы было понятно, какие можно удалять.

## Release 4: Безопасность и тесты

Проверь свои эндпоинты согласно тестам, запустить их можно на сервере с помощью `npm test`. Защити эндпоинты, которые этого требуют:
- параметризированные эндпоинты должны проверять валидность параметра
- совершение CRUD действий должно быть разрешено по токену доступа
- некоторые действия (например, удаление и редактирование резюме) должны быть доступны только автору резюме

Проверь, что зарегистированный пользователь не сможет заново перейти на страницу регистрации. А гость не сможет войти в личный кабинет.

## Release 5: Поиск

Интегрируй поиск по ключевым словам и фильтрам. Например, чтобы через поисковую строку можно было найти резюме по ключевым словам, содержанию или тэгам.

## Release 6*: Аналитика и отчеты

Какое-то резюме часто смотрят? Давай соберём статистику и покажем её владельцу! Интегрируй аналитику для отслеживания активности пользователей и популярности резюме.

Учти следующие требования:
- Сбор и отображение статистики по просмотрам резюме.
- Отчеты о популярности определенных навыков и опыта работы.
- Визуализация данных с помощью графиков и диаграмм.
