# LangChain RAG demo (Node.js + TypeScript + tsx)

Демо-проект по RAG на LangChain (JS/TS)

## Что внутри

- **Индексация**: чтение `data/*.txt` → чанкинг → embeddings → `MemoryVectorStore`
- **Embeddings**: `OpenAIEmbeddings` (с поддержкой OpenAI-совместимого прокси через `baseURL`)
- **Векторное хранилище**: `MemoryVectorStore` из `@langchain/classic` (in-memory демо)
- **Два сценария**:
  - `src/demo-rag-agent.ts`: RAG-агент с tool `retrieve` (видно шаги в streaming)
  - `src/demo-rag-chain.ts`: упрощённый RAG-chain (retrieval + 1 вызов LLM)

## Запуск

1) Установить зависимости:

```bash
npm i
```

2) Создать `.env`:

```bash
copy .env.example .env
```

3) Настроить переменные окружения в `.env`:

- **`OPENAI_API_KEY`**: ключ провайдера (для Polza обычно начинается с `pza_...`, для OpenAI — `sk-...`)
- **`OPENAI_BASE_URL`** (опционально): OpenAI-совместимый endpoint (например `https://polza.ai/api/v1`)
  - также поддерживается **`BASE_URL`** (для совместимости)
- **`OPENAI_MODEL`** (опционально): модель для chat (зависит от провайдера/прокси)

4) Запуск:

```bash
npm run dev
```

Запуск chain-варианта:

```bash
npm run dev:chain
```

## Чем отличаются `npm run dev` и `npm run dev:chain`

В `package.json` есть два скрипта:

- **`npm run dev`** → запускает файл `src/demo-rag-agent.ts`
- **`npm run dev:chain`** → запускает файл `src/demo-rag-chain.ts`

### `npm run dev` (RAG agent)

- **Что это**: агент, который умеет вызывать инструмент `retrieve` для поиска релевантных фрагментов.
- **Как работает**:
  - при вопросе агент сам решает, когда вызвать `retrieve({ query })`
  - `retrieve` делает `vectorStore.similaritySearch()` и возвращает контекст + “артефакт” (найденные документы)
  - агент использует контекст и формирует финальный ответ
- **Сколько вызовов LLM**: обычно **1+** (агент может делать дополнительные шаги в зависимости от поведения).
- **Вывод в консоль**: включён **streaming шагов**, поэтому видно промежуточные сообщения/шаги.
- **Когда выбирать**: когда хочешь увидеть “агентный” подход из документации LangChain RAG и посмотреть, как tool-использование выглядит в реальном времени.

### `npm run dev:chain` (упрощённый RAG chain)

- **Что это**: простой и быстрый сценарий “retrieval → один ответ LLM”.
- **Как работает**:
  - код сам делает `vectorStore.similaritySearch(question, k)`
  - склеивает `CONTEXT` из top-k чанков
  - делает **один** `model.invoke(...)` и печатает результат
- **Сколько вызовов LLM**: ровно **1** (на один пользовательский вопрос).
- **Вывод в консоль**: только финальный ответ (без стриминга шагов).
- **Когда выбирать**: когда нужна максимальная простота и предсказуемость (быстрый демо-вариант без “агентности”).

Можно передать вопрос аргументом:

```bash
npm run dev -- "Что сказано про собаку?"
```

```bash
npm run dev:chain -- "О чём договор и какие там условия оплаты?"
```
