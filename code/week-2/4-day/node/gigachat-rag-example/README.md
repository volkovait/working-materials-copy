# GigaChat RAG

Демо‑проект **RAG (Retrieval Augmented Generation)** на Node.js + Express с использованием пакета [`gigachat`](https://www.npmjs.com/package/gigachat).

## Что делает проект

- **Загрузка файлов** через браузер (эндпоинт `POST /upload`)
- **Сохранение файлов** на диск в папку `uploads/`
- **Индексация текста**: текстовые файлы режутся на фрагменты (чанки), для каждого чанка считаются **эмбеддинги** через GigaChat и сохраняются в in‑memory векторное хранилище
- **Чат** (эндпоинт `POST /chat`):
  - для каждого вопроса пользователя считаем эмбеддинг вопроса
  - находим top‑K наиболее похожих чанков (cosine similarity)
  - добавляем эти выдержки в промпт модели и просим отвечать по контексту

## Быстрый старт

1) Установить зависимости:

```bash
npm i
```

2) Создать файл `.env` в корне проекта:

```env
GIGACHAT_API_KEY=<ваши_учётные_данные>
PORT=3000
```

3) Запустить сервер:

```bash
npm start
```

4) Открыть в браузере:

- `http://localhost:3000`

## Как пользоваться (сценарий демо)

1) Открой страницу.
2) Загрузи один или несколько **текстовых** файлов (см. “Поддерживаемые форматы”).
3) В чате задай вопрос по содержимому загруженных файлов.
4) (Опционально) удали файл из списка — индекс будет пересобран из оставшихся файлов.

## Поддерживаемые форматы для индексации

Индексируются только текстовые файлы:

- по `mimetype`: `text/*`
- или по расширению: `.txt`, `.md`, `.markdown`, `.csv`, `.tsv`, `.json`, `.log`

Любые другие файлы будут **сохранены** в `uploads/`, но **не будут проиндексированы**.

## Важные технические моменты (как устроен RAG)

### 1) Загрузка файлов (Multer)

- Настройка Multer находится в `src/config/serverConfig.js`
- Файлы сохраняются в `uploads/` (папка создаётся автоматически)

### 2) Гидратация состояния из `uploads/`

При старте процесса проект:

- читает `uploads/`
- восстанавливает список файлов в памяти процесса
- пересобирает RAG‑индекс из текстовых файлов

Это реализовано в `src/utils/uploadsRagState.js` и вызывается через `FileService.hydrate()`.

### 3) Индексация (чанки + эмбеддинги)

Файл режется на чанки с overlap:

- `maxLen = 1200`
- `overlap = 200`

Код находится в `src/utils/vectorStore.js`:

- `addFileToVectorStore()` — добавляет документ(ы) в индекс
- `rebuildVectorStoreFromTextFiles()` — пересобирает индекс из набора файлов

### 4) Retrieval + Generation

При запросе в чат:

1. `retrieveRelevant(message, k)` возвращает top‑K чанков
2. Эти чанки добавляются в контекст промпта
3. Далее вызывается `gigaChatClient.chat(...)`

Код:

- `src/services/Chat.service.js` — сборка контекста и вызов модели
- `src/utils/vectorStore.js` — retrieval по cosine similarity

## Архитектура: routes → controllers → services → utils

Проект разделён на слои:

- `src/routes/*` — только роутинг (какой URL вызывает какой контроллер)
- `src/controllers/*` — обработка HTTP‑запросов/ответов (валидация, статусы, json)
- `src/services/*` — бизнес‑логика (индексация, удаление, запрос к модели)
- `src/utils/*` — низкоуровневые утилиты (векторное хранилище, гидратация uploads, клиент gigachat)

## Ограничения демо (важно)

- **Векторный индекс хранится в памяти процесса** (MemoryVectorStore). После перезапуска он пересобирается из `uploads/`, но на больших объёмах данных это будет долго.
- **Нет парсеров PDF/DOCX**: бинарные/сложные форматы не поддержаны.
- **TLS**: в `src/utils/gigachat.js` включён `rejectUnauthorized: false` (удобно для учебной среды, но не для production).

## Типовые проблемы

- **Статика не грузится**: убедись, что папка `src/public` существует и сервер запущен из корня проекта.
- **Ошибка GigaChat / embeddings**: проверь `GIGACHAT_API_KEY` в `.env`.
- **Файл не индексируется**: проверь расширение и что он действительно текстовый (UTF‑8).
