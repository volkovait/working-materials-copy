## Шаг 1: Установка веб-сервера nginx

nginx — это веб-сервер, который будет принимать запросы от пользователей и перенаправлять их на n8n, работающий в Docker контейнере.

- `sudo` — команда для выполнения с правами администратора
- `apt install` — установка пакета через менеджер пакетов Ubuntu

```bash
# Выполнение команды с правами администратора
sudo apt install nginx
```

## Шаг 2: Создание конфигурационного файла для вашего домена

Эта команда откроет текстовый редактор nano для создания файла конфигурации. Замените `<your_domain>` на ваш реальный домен (например: `example.com`).

Файл будет содержать настройки, как nginx должен обрабатывать запросы к вашему домену.

- `/etc/nginx/sites-available/` — директория, где хранятся доступные конфигурации сайтов

```bash
# Выполнение команды с правами администратора
# Открытие текстового редактора nano для создания/редактирования конфигурационного файла
# Замените <your_domain> на ваш реальный домен
sudo nano /etc/nginx/sites-available/<your_domain>
```

```bash
# Начало блока конфигурации для HTTPS сервера
server {
    # Указание доменного имени, для которого применяется эта конфигурация
    server_name 5xai.ru;

    # Начало блока настройки обработки всех запросов (location /)
    location / {
      # Перенаправление всех запросов на локальный сервер n8n на порту 5678
      proxy_pass http://localhost:5678;
      # Передача оригинального заголовка Host клиента в запрос к n8n
      proxy_set_header Host $host;
      # Передача реального IP-адреса клиента в заголовке X-Real-IP
      proxy_set_header X-Real-IP $remote_addr;
      # Передача цепочки IP-адресов прокси-серверов в заголовке X-Forwarded-For
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      # Передача протокола соединения (http/https) в заголовке X-Forwarded-Proto
      proxy_set_header X-Forwarded-Proto $scheme;      
      # Установка версии HTTP протокола 1.1 для поддержки WebSocket соединений
      proxy_http_version 1.1;
      # Передача заголовка Upgrade для переключения на WebSocket протокол
      proxy_set_header Upgrade $http_upgrade;
      # Установка заголовка Connection со значением "upgrade" для WebSocket соединений
      proxt_set_header Connection "upgrade"
    }

    # Прослушивание HTTPS соединений на IPv6 адресе порта 443 с SSL
    listen [::]:443 ssl ipv6only=on; # managed by Certbot
    # Прослушивание HTTPS соединений на IPv4 адресе порта 443 с SSL
    listen 443 ssl; # managed by Certbot
    # Путь к SSL сертификату (полная цепочка сертификатов)
    ssl_certificate /etc/letsencrypt/live/5xai.ru/fullchain.pem; # managed by Certbot
    # Путь к приватному ключу SSL сертификата
    ssl_certificate_key /etc/letsencrypt/live/5xai.ru/privkey.pem; # managed by Certbot
    # Подключение стандартных настроек SSL для nginx от Certbot
    include /etc/letsencrypt/options-ssl-nginx.conf; # managed by Certbot
    # Путь к файлу параметров Диффи-Хеллмана для усиления безопасности SSL
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; # managed by Certbot
}

# Начало блока конфигурации для HTTP сервера (перенаправление на HTTPS)
server {
    # Условие: если запрос приходит на домен 5xai.ru
    if ($host = 5xai.ru) {
        # Перенаправление на HTTPS версию того же адреса (301 - постоянное перенаправление)
        return 301 https://$host$request_uri;
    } # managed by Certbot


    # Прослушивание HTTP соединений на IPv4 адресе порта 80
    listen 80;
    # Прослушивание HTTP соединений на IPv6 адресе порта 80
    listen [::]:80;
    # Указание доменного имени для этого блока
    server_name 5xai.ru;
    # Возврат ошибки 404 для всех остальных запросов (не подходящих под условие выше)
    return 404; # managed by Certbot
}
```

**Важно:** В открывшемся редакторе вставьте конфигурацию nginx (см. README.md, раздел "Настройка nginx для HTTPS"). После вставки нажмите `Ctrl+O` для сохранения, затем `Enter` для подтверждения, и `Ctrl+X` для выхода.

## Шаг 3: Активация конфигурации сайта

Эта команда создает символическую ссылку (ярлык) из директории `sites-available` в `sites-enabled`. nginx использует только конфигурации из `sites-enabled`.

- `ln -s` — создание символической ссылки (symbolic link)

Таким образом, ваш сайт становится активным для nginx.

```bash
# Выполнение команды с правами администратора
# Создание символической ссылки (ln -s) из директории sites-available в sites-enabled
# Замените <your_domain> на ваш реальный домен
sudo ln -s /etc/nginx/sites-available/<your_domain> /etc/nginx/sites-enabled/
```

## Шаг 4: Проверка корректности конфигурации nginx

Перед перезапуском nginx важно проверить, что в конфигурации нет ошибок.

- `-t` означает "test" (тест)

Если есть ошибки, команда покажет их, и вы сможете исправить перед перезапуском. Если все хорошо, увидите сообщение "syntax is ok" и "test is successful".

```bash
# Выполнение команды с правами администратора
# Проверка синтаксиса конфигурации nginx (-t означает test)
sudo nginx -t
```

## Шаг 5: Перезапуск веб-сервера nginx

После проверки конфигурации нужно перезапустить nginx, чтобы применить изменения.

- `systemctl` — системный менеджер для управления службами в Linux
- `restart` — перезапуск службы

После этой команды nginx начнет использовать новую конфигурацию.

```bash
# Выполнение команды с правами администратора
# Перезапуск службы nginx через systemctl (системный менеджер служб)
sudo systemctl restart nginx
```

## Шаг 6: Установка Certbot для получения SSL сертификатов

Certbot — это инструмент для автоматического получения и обновления SSL сертификатов от Let's Encrypt. SSL сертификаты нужны для безопасного HTTPS соединения (защищенное соединение).

- `python3-certbot-nginx` — плагин для работы с nginx

Без SSL сертификата браузеры будут показывать предупреждение о небезопасном соединении.

```bash
# Выполнение команды с правами администратора
# Установка certbot (инструмент для получения SSL сертификатов) и плагина для nginx
sudo apt install certbot python3-certbot-nginx
```

## Шаг 7: Получение SSL сертификата для вашего домена

Эта команда автоматически:

1. Проверяет, что домен указывает на ваш сервер
2. Получает SSL сертификат от Let's Encrypt (бесплатный)
3. Настраивает nginx для использования HTTPS

- `--nginx` — указывает, что нужно настроить nginx автоматически
- `-d <your_domain>` — указывает домен, для которого нужен сертификат

**Важно:** Замените `<your_domain>` на ваш реальный домен (например: `-d example.com`). Certbot может задать несколько вопросов — отвечайте согласно инструкциям на экране.

```bash
# Выполнение команды с правами администратора
# Запуск certbot с автоматической настройкой nginx (--nginx)
# Получение SSL сертификата для указанного домена (-d <your_domain>)
# Замените <your_domain> на ваш реальный домен
sudo certbot --nginx -d <your_domain>
```

## Шаг 8: Запуск n8n в Docker контейнере с поддержкой HTTPS

Эта команда запускает n8n в изолированном контейнере Docker.

**Параметры команды:**

- `docker run` — запуск нового контейнера
- `-d` — запуск в фоновом режиме (detached mode), контейнер будет работать в фоне
- `--rm` — автоматическое удаление контейнера при его остановке (экономит место)
- `--name n8n` — имя контейнера для удобного управления
- `-p 5678:5678` — проброс портов (внешний:внутренний), порт 5678 — стандартный порт n8n
- `-v n8n_data:/home/node/.n8n` — монтирование тома для сохранения данных n8n (workflows, credentials)
- `-v /etc/letsencrypt:/etc/letsencrypt` — монтирование директории с SSL сертификатами в контейнер
- `-e` — установка переменных окружения:
  - `N8N_SECURE_COOKIE=true` — использование защищенных cookies (требуется для HTTPS)
  - `N8N_HTTPS_CERT` — путь к SSL сертификату внутри контейнера
  - `N8N_HTTPS_KEY` — путь к приватному ключу SSL сертификата
- `docker.n8n.io/n8nio/n8n:stable` — образ Docker с n8n (стабильная версия)

**Важно:** Замените `<your_domain>` на ваш реальный домен в путях к сертификатам.

```bash
# Запуск нового Docker контейнера
docker run -d --rm \
# Запуск в фоновом режиме (-d) и автоматическое удаление при остановке (--rm)
--name n8n \
# Задание имени контейнера для удобного управления
-p 5678:5678 \
# Проброс порта 5678 (внешний:внутренний) для доступа к n8n
-v n8n_data:/home/node/.n8n \
# Монтирование тома n8n_data для сохранения данных n8n
-v /etc/letsencrypt:/etc/letsencrypt \
# Монтирование директории с SSL сертификатами в контейнер
-e N8N_SECURE_COOKIE=true \
# Включение использования защищенных cookies (требуется для HTTPS)
-e N8N_HTTPS_CERT=/etc/letsencrypt/live/<your_domain>/fullchain.pem \
# Установка пути к SSL сертификату (замените <your_domain> на ваш домен)
-e N8N_HTTPS_KEY=/etc/letsencrypt/live/<your_domain>/privkey.pem \
# Установка пути к приватному ключу SSL сертификата (замените <your_domain> на ваш домен)
docker.n8n.io/n8nio/n8n:stable
# Образ Docker с n8n (стабильная версия)
```

## После выполнения всех шагов

После выполнения всех шагов:

- n8n будет доступен по адресу `https://<your_domain>`
- Все соединения будут защищены SSL сертификатом
- Данные n8n будут сохраняться в Docker томе `n8n_data`

**Полезные команды для управления контейнером:**

- `docker stop n8n` — остановка контейнера
- `docker start n8n` — запуск контейнера
- `docker logs n8n` — просмотр логов
