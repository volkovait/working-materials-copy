## Что такое n8n

**n8n** — это платформа автоматизации рабочих процессов с открытым исходным кодом. Она позволяет соединять разные сервисы и приложения через визуальный интерфейс: перетаскивать блоки, настраивать логику и запускать автоматические цепочки действий — без написания кода (или с минимальным кодом, если нужно).

Простой пример: пришло письмо → ИИ анализирует его → результат отправляется в Telegram. Всё это можно настроить за 10 минут.

## Базовые сущности

### Workflow (Рабочий процесс)

Это центральная единица n8n — **визуальная схема** из соединённых блоков. Каждый workflow описывает одну автоматизацию: что запускает процесс, что он делает и куда отправляет результат.

- Хранится в базе данных n8n
- Может быть включён (`active`) или выключен
- Запускается триггером или вручную

### Node (Узел)

Каждый блок на холсте — это узел. Узел выполняет одно конкретное действие. Узлы соединяются стрелками: данные «текут» от одного к другому.

Узлы делятся на несколько видов:

| Вид | Описание | Примеры |
|-----|----------|---------|
| **Trigger** (триггер) | Запускает workflow при наступлении события | Chat Trigger, Telegram Trigger, Schedule, Webhook, Form Trigger |
| **Action** (действие) | Выполняет операцию во внешнем сервисе | Отправить сообщение в Telegram, сохранить в Google Sheets |
| **AI** | Работа с языковыми моделями и агентами | AI Agent, OpenAI Chat Model, Groq Chat Model |
| **Transform** (трансформация) | Обработка данных | Code (JS/Python), Set, If, Merge, Split |
| **Tool** (инструмент ИИ) | Даёт агенту возможность выполнять действия | HTTP Request Tool, Wikipedia, Calculator |
| **Memory** (память) | Хранит историю диалога для ИИ-агента | Simple Memory (Buffer Window) |

### Credentials (Учётные данные)

Ключи доступа к внешним сервисам (API-ключи, токены, логины). Хранятся зашифрованно внутри n8n. Настраиваются один раз и переиспользуются во всех workflows.

**Где настраивать:** левое меню → Credentials → Add Credential.

### Execution (Запуск)

Каждый запуск workflow создаёт запись `Execution` — лог с входными и выходными данными каждого узла. Незаменим при отладке.

**Где смотреть:** левое меню → Executions.

### Item (Элемент данных)

Единица данных, которая передаётся между узлами. Узлы всегда работают с массивом items. Например, если HTTP-запрос вернул список из 5 новостей — это 5 items, которые следующий узел обработает по очереди.

### Expression (Выражение)

Динамическое значение в поле узла. Пишется в двойных фигурных скобках:

```
{{ $json["email"] }}
{{ $node["AI Agent"].json["output"] }}
{{ $fromAI('city', '', 'string') }}
```

## Как работать с Workflow

### Шаг 1 — Открыть редактор

1. Перейдите в n8n (например `http://localhost:5678` или ваш домен)
2. Нажмите **+ New workflow** или откройте существующий

### Шаг 2 — Добавить триггер

Каждый workflow начинается с триггера. Кликните на `+` в центре холста и выберите тип:
- `When chat message received` — чат-интерфейс прямо в n8n
- `Telegram Trigger` — сообщение в Telegram-боте
- `Schedule Trigger` — по расписанию (каждый час, каждый день и т.д.)
- `Webhook` — HTTP-запрос извне

### Шаг 3 — Добавить узлы действий

Кликните на `+` в конце стрелки от триггера и добавляйте нужные узлы. Они выполняются последовательно.

### Шаг 4 — Настроить узлы

Кликните на узел → откроется панель настройки. Заполните поля. Если поле поддерживает Expressions, нажмите иконку `=` рядом с полем и вставьте выражение.

### Шаг 5 — Протестировать

Нажмите **Execute workflow** (кнопка внизу). Данные пройдут через все узлы, вы увидите результат на каждом шаге прямо на холсте.

### Шаг 6 — Включить

Переключите тумблер **Active** в правом верхнем углу. Теперь workflow будет запускаться автоматически.

### Импорт готового workflow

Если у вас есть `.json` файл:
1. Откройте n8n → **Workflows**
2. Нажмите **Import from file** или **Import from URL**
3. Выберите `.json` файл
4. Настройте Credentials (API-ключи) для каждого узла, требующего авторизации
