# Сервис погоды MCP

Этот проект представляет собой сервер Model Context Protocol (MCP) для получения данных о погоде, использующий API Национальной метеорологической службы США (NWS).

## Возможности

- Получение текущих погодных предупреждений для указанного штата США
- Получение прогноза погоды для указанных координат на территории США

## Требования

- Node.js версии 16 или выше
- npm или yarn для управления зависимостями
- Доступ к интернету для работы с API NWS

## Установка

1. Клонируйте репозиторий:
```bash
git clone [url-репозитория]
cd weather-mcp
```

2. Установите зависимости:
```bash
npm install
```

3. Соберите проект:
```bash
npm run build
```

## Использование

### Запуск сервера

```bash
npm start
```

### Доступные инструменты

#### 1. Получение погодных предупреждений (get-alerts)

Позволяет получить активные погодные предупреждения для указанного штата США.

Параметры:
- `state`: Двухбуквенный код штата (например, CA для Калифорнии, NY для Нью-Йорка)

#### 2. Получение прогноза погоды (get-forecast)

Позволяет получить прогноз погоды для указанных координат на территории США.

Параметры:
- `latitude`: Широта местоположения (от -90 до 90)
- `longitude`: Долгота местоположения (от -180 до 180)

## Подключение к Cursor

Сервер использует stdio-транспорт и подключается через конфигурационный файл `mcp.json`.

### Способ 1: Проектный уровень (рекомендуется)

Создайте файл `.cursor/mcp.json` в корне проекта, в котором хотите использовать сервер:

```json
{
  "mcpServers": {
    "weather": {
      "command": "node",
      "args": ["/абсолютный/путь/до/weather-mcp/build/index.js"]
    }
  }
}
```

Пример для Windows:
```json
{
  "mcpServers": {
    "weather": {
      "command": "node",
      "args": ["C:\\Users\\username\\weather-mcp\\build\\index.js"]
    }
  }
}
```

### Способ 2: Глобальный уровень (для всех проектов)

Отредактируйте файл глобальных настроек MCP:

- **Windows:** `%APPDATA%\Cursor\User\mcp.json`
- **macOS/Linux:** `~/.cursor/mcp.json`

Добавьте запись в объект `mcpServers`:

```json
{
  "mcpServers": {
    "weather": {
      "command": "node",
      "args": ["/абсолютный/путь/до/weather-mcp/build/index.js"]
    }
  }
}
```

### Способ 3: Через npm run start

Если вы не хотите указывать путь до скомпилированного файла, можно запускать через npm с указанием рабочей директории:

```json
{
  "mcpServers": {
    "weather": {
      "command": "npm",
      "args": ["run", "start"],
      "cwd": "/абсолютный/путь/до/weather-mcp"
    }
  }
}
```

### Проверка подключения

1. Убедитесь, что проект собран: `npm run build`
2. Перезапустите Cursor или выполните команду `MCP: Restart All Servers` в палитре команд (`Ctrl+Shift+P`)
3. Откройте настройки Cursor → Features → MCP и убедитесь, что сервер `weather` отображается со статусом **connected**
4. Протестируйте в чате Cursor Agent:
   - _"Получи погодные предупреждения для штата Калифорния (CA)"_
   - _"Какой прогноз погоды для координат 40.7128, -74.0060?"_

## Ограничения

- Сервис работает только с локациями на территории США
- Требуется стабильное интернет-соединение для работы с API
- API NWS может иметь ограничения на количество запросов
