# Swarm-архитектура и суммаризация контекста

Учебный пример мультиагентной системы на **LangGraph JS** с архитектурой **Swarm** и демонстрацией **суммаризации контекста** для управления длинной историей сообщений.

## Что сделано

- Агенты передают управление друг другу через `createHandoffTool` из `@langchain/langgraph-swarm`, без центрального супервизора.
- **Суммаризация контекста:** при росте истории диалога старые сообщения сжимаются в краткое саммари (стратегия «сжатие в саммари»), чтобы укладываться в лимит контекста модели и снижать затраты.

## Зависимости и запуск

```bash
npm install
# В .env задать OPENAI_API_KEY (и при необходимости OPENAI_BASE_URL)
npm start
```

По умолчанию скрипт: изучает текущую директорию, читает `package.json`, создаёт файл `SUMMARY.md` с отчётом о проекте. Агенты **researcher** и **writer** координируются через handoff.

## Архитектура Swarm

- Используются **createSwarm** и **createHandoffTool** из [@langchain/langgraph-swarm](https://reference.langchain.com/javascript/langchain-langgraph-swarm).
- Агенты собираются через **createReactAgent** из `@langchain/langgraph/prebuilt` (скомпилированный граф с узлом `tools` нужен для `getHandoffDestinations` внутри Swarm; `createAgent` из пакета `langchain` этой совместимости не даёт).
- **researcher** — читает директории и файлы (`list_directory`, `read_file`); при необходимости записать файл передаёт управление **writer** через handoff.
- **writer** — пишет файлы (`write_file`); при необходимости прочитать данные передаёт управление **researcher**.
- **defaultActiveAgent: "researcher"** — первый ответ на запрос пользователя даёт исследователь.
- Состояние диалога сохраняется в **MemorySaver** (checkpointer) по `thread_id`, что позволяет вести многошаговые сессии.

Отличие от Supervisor: нет отдельного «менеджера», который решает, кому передать задачу; агенты сами вызывают handoff-инструмент и передают управление коллеге.

## Суммаризация контекста

- **Ограничения контекста:** у модели ограничен объём ввода; на каждом шаге нужна релевантная часть истории, а не вся подряд.
- **Стратегия «сжатие в саммари»:** старая часть истории не отдаётся целиком — она пересказывается одним кратким сообщением, последние сообщения остаются без изменений.

В коде:

- Константа **MAX_MESSAGES_BEFORE_SUMMARY** задаёт порог (по умолчанию 6 сообщений).
- Функция **summarizeContextIfNeeded(messages)**:
  - если сообщений не больше порога — возвращает массив как есть;
  - иначе берёт все, кроме последних N сообщений, отправляет их в LLM с промптом «сократи до ключевых фактов и решений», подставляет в начало одно системное сообщение с саммари и добавляет последние сообщения без изменений.
- В длинной сессии (много обращений в одном `thread_id`) можно вызывать суммаризацию перед очередным вызовом графа, передавая уже «поджатую» историю в `messages`.

Так мы явно ограничиваем объём контекста и демонстрируем практику **менеджмента состояния и контекста** в мультиагентных системах.

## Демонстрация суммаризации

При одном запросе в начале сессии суммаризация обычно не срабатывает (мало сообщений). Чтобы увидеть срабатывание:

- Вариант 1: вести многошаговый диалог в одном `thread_id` и перед каждым вызовом вызывать `summarizeContextIfNeeded` над накопленными `messages` (логику можно вынести в цикл или middleware).
- Вариант 2: в `index.js` раскомментировать блок с `DEMO_SUMMARIZATION` и при необходимости задать переменную окружения `DEMO_SUMMARIZATION=1`, чтобы искусственно добавить длинную историю и увидеть сжатие перед запуском Swarm.

## LangSmith (трейсинг и отладка)

В этом репозитории для демонстрации трейсинга и отладки мультиагентных запусков используется **LangSmith**. См. проект **langsmith-supervisor-architecture-example** в той же папке `node/`.

**Зачем нужен LangSmith:**

- **Трейсинг запусков** — каждый прогон графа (все агенты и вызовы инструментов) виден в одном trace в веб-интерфейсе LangSmith.
- **Отладка** — просмотр входов/выходов каждого шага (промпты, ответы LLM, результаты инструментов), что упрощает поиск ошибок и странного поведения.
- **Метрики** — оценка времени выполнения и расхода токенов по запускам.

**Как подключить в этом проекте:** задать в `.env` (в `code/week-4/11-day/node/`) переменные `LANGSMITH_TRACING=true`, `LANGSMITH_API_KEY` и при необходимости `LANGSMITH_ENDPOINT`, `LANGSMITH_PROJECT`. Дополнительных зависимостей не требуется — поддержка встроена в LangChain/LangGraph. После этого запуски этого Swarm-приложения тоже будут отправляться в выбранный проект LangSmith.
