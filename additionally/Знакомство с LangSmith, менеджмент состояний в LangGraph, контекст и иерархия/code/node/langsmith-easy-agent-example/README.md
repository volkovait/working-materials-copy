# Пример LangSmith-демо на LangGraph JS

Учебный проект для **демонстрации LangSmith**: один агент с двумя инструментами (`get_greeting`, `write_file`) без супервизора и handoff. Так сохраняется корректный порядок сообщений для API и стабильно работают трейсы в LangSmith.

## Описание примера

- **Один агент** (ReAct) с инструментами:
  - `get_greeting` — возвращает короткое приветствие;
  - `write_file` — записывает текст в файл.
- **Задача:** получить приветствие и записать его в `greeting.txt`. Цепочка короткая, без риска `GRAPH_RECURSION_LIMIT` и без ошибки «tool_calls без ответов» на кастомных API (handoff убран).

## Зачем добавлен LangSmith

**LangSmith** — платформа мониторинга и отладки для приложений на LangChain/LangGraph. В этом проекте он используется для:

1. **Трейсинга запусков** — каждый вызов графа отправляется в LangSmith как один trace. В интерфейсе видна цепочка: запрос пользователя → вызовы инструментов агента → ответ агента (супервизор в этом примере не используется).
2. **Просмотра входов/выходов** — для каждого шага видны промпты, ответы модели и результаты инструментов, что упрощает отладку и поиск причин ошибок.
3. **Анализа задержек и затрат** — по каждому запуску можно оценить время выполнения и использование токенов.

Дополнительный код в приложении не требуется: при наличии переменных окружения LangSmith трейсинг включается автоматически (поддержка встроена в `@langchain/core` и LangGraph).

## Какие переменные окружения нужны

Все настройки берутся из файла `.env`:

| Переменная | Назначение |
|-----------|------------|
| `OPENAI_API_KEY` | Ключ API для модели (обязательно). |
| `OPENAI_BASE_URL` | Опционально: кастомный базовый URL API. |
| `LANGSMITH_TRACING` | `true` — включить отправку трейсов в LangSmith. |
| `LANGSMITH_API_KEY` | Ключ API LangSmith (нужен для трейсинга). |
| `LANGSMITH_ENDPOINT` | URL API LangSmith (по умолчанию `https://api.smith.langchain.com`). |
| `LANGSMITH_PROJECT` | Имя проекта в LangSmith, куда попадают трейсы. |

Если `LANGSMITH_TRACING` не `true` или нет `LANGSMITH_API_KEY`, приложение работает без отправки трейсов в LangSmith (агент выполняется локально).

## Как запустить

1. Установить зависимости:
   ```bash
   npm install
   ```

2. В `.env` задать как минимум `OPENAI_API_KEY`. Файл `.env` загружается из **родительской папки** (`node/.env`), а не из папки примера. Для LangSmith добавить `LANGSMITH_TRACING=true`, `LANGSMITH_API_KEY` и при необходимости `LANGSMITH_PROJECT`.

3. Запустить:
   ```bash
   npm start
   ```

В консоли выводится лог шагов агента; при включённом LangSmith запуск появится в выбранном проекте на [smith.langchain.com](https://smith.langchain.com). В папке примера создаётся файл `greeting.txt`.

## Файлы

- `index.js` — точка входа: загрузка `.env` из родительской папки, модель, два инструмента, один агент на **createReactAgent** (@langchain/langgraph/prebuilt), одна короткая задача для демо LangSmith.
- `package.json` — зависимости; отдельный пакет для LangSmith не требуется.
